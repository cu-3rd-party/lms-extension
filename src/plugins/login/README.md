# Плагин `login` — страница входа

**URL:** страница входа в LMS (форма авторизации)

Исправляет поведение браузерного автодополнения на форме входа.

---

## Файлы и их назначение

### `fix_email_autocomplete.js`

**Проблема:** Поле ввода логина на странице входа в LMS не имеет корректных атрибутов для автодополнения. Браузер не распознаёт его как поле email и не предлагает сохранённые учётные записи.

**Как решается:**

1. `MutationObserver` ждёт появления `input[name="userName"]` в DOM.
2. Устанавливает атрибуты:
   - `type="email"` — браузер распознаёт поле как email
   - `autocomplete="username"` — стандартное значение для логина
   - `name="username"` — стандартное имя поля
3. Оборачивает input в `<form autocomplete="on">`, если formы ещё нет.
4. Добавляет скрытый `<input type="password" autocomplete="current-password">` внутрь формы — большинство браузеров требуют наличия поля пароля в форме для активации автодополнения.

**Взаимодействие со страницей:**

- Читает DOM: `input[name="userName"]`
- Изменяет: атрибуты существующего `<input>`
- Вставляет: `<form>` и скрытый `<input type="password">` (если их нет)
- Fetch-запросов не делает

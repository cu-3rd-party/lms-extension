# Плагин `course-view` — страница курсов

**URL:** `/learn/courses/view` (список курсов, обзор конкретного курса)

Набор скриптов для улучшения работы со списком курсов и страницей отдельного курса.

---

## Файлы и их назначение

### `courses_fix.js`

Центральный скрипт плагина. Управляет архивацией курсов, реагирует на SPA-навигацию и координирует запуск остальных скриптов.

**Как работает:**

1. Подписывается на `MutationObserver` за изменением URL (SPA-навигация) и перезапускает логику при переходе между страницами.
2. Добавляет на каждой карточке курса кнопку-архивации (иконка архива). При клике курс скрывается и его ID сохраняется в `browser.storage.sync` → `archivedCourseIds`.
3. Если в попапе включён «Старый дизайн» — добавляет CSS-классы для drag-and-drop курсов (`cursor: grab`).
4. При изменении ключевых настроек (`oldCoursesDesignToggle`, `futureExamsViewToggle`, `courseOverviewTaskStatusToggle`) перезагружает страницу.
5. Запускает `simplifyAllCourseCards()`, `activateCourseOverviewTaskStatus()`, `activateCourseOverviewAutoscroll()` и `viewFutureExams()` по условию из настроек.

**Взаимодействие со страницей:**

- Читает DOM: `ul.course-list`, карточки `cu-course-card`
- Вставляет: кнопки-архиватора, CSS-стили для drag-and-drop
- Хранилище: `browser.storage.sync` (archivedCourseIds, настройки)

---

### `course_card_simplifier.js`

Заменяет стандартные карточки курсов на компактные текстовые плашки.

**Как работает:**

1. Находит все `li.course-list__item` в `ul.course-list`.
2. Внутри каждой карточки создаёт `<div class="simplified-course-card">` с названием курса.
3. Скрывает оригинальное содержимое `cu-course-card`, оставляя только созданный div.
4. Если название слишком длинное — обрезает его с многоточием (порог зависит от ширины экрана: мобайл/планшет/десктоп).
5. Кнопка архива (из `courses_fix.js`) переносится в новую плашку.
6. Тема (тёмная/светлая) читается из хранилища и применяется к стилям.

**Взаимодействие со страницей:**

- Читает DOM: `ul.course-list`, `.course-name`, `.archive-button-container`
- Вставляет: `<div class="simplified-course-card">` внутрь каждой карточки

---

### `course_overview_task_status.js`

Отображает статус задачи прямо в раскрывающемся аккордеоне курса.

**Как работает:**

1. Извлекает `courseId` из URL (`/view/actual/123` или `/view/archived/123`).
2. Параллельно запрашивает три API-эндпоинта LMS:
   - `/api/micro-lms/courses/{id}/exercises` — список упражнений
   - `/api/micro-lms/courses/{id}/student-performance` — успеваемость
   - `/api/micro-lms/tasks/student` — список задач с датами
3. Строит карту `longreadId → статус последней задачи`.
4. Для каждого `tui-accordion-item` в `cu-course-overview` ищет соответствующий лонгрид и вставляет иконку статуса (✅ сдано / ❌ просрочено / ⏩ скип / 🔄 доработка / ⏳ ожидает).

**Взаимодействие со страницей:**

- Читает DOM: `cu-course-overview`, `tui-accordion-item`
- Вставляет: иконки/чипы статуса рядом с каждой темой аккордеона
- Fetch: 3 API-запроса к LMS

---

### `course_overview_autoscroll.js`

Автоматически прокручивает к последней теме курса при открытии.

**Как работает:**

1. Ждёт появления `cu-course-overview` (через `waitForElement` с таймаутом 10 сек).
2. Берёт все `tui-accordion-item`, которые **не** являются будущими экзаменами (`data-item-type !== 'future-exam'`).
3. Прокручивает к последнему из них (`scrollIntoView({ behavior: 'smooth', block: 'center' })`).
4. Кликает по нему, чтобы раскрыть.

**Взаимодействие со страницей:**

- Читает DOM: `cu-course-overview`, `tui-accordion-item`
- Вызывает: `scrollIntoView()`, `button.click()`

---

### `course_friends_list.js`

Виджет с информацией о том, кто из друзей также записан на открытый курс.

**Как работает:**

1. Читает список друзей из `localStorage` (`cu_friends_list`).
2. Запрашивает у LMS API данные о курсах для каждого друга.
3. Нормализует названия курсов и сравнивает с текущим (учитывает разные форматы написания, C++, etc.).
4. Создаёт плавающий виджет (`<div id="cu-friends-floating-widget">`) в правом нижнем углу страницы.
5. В виджете отображается список друзей: тех, кто на том же курсе — в начале, остальные — ниже.
6. Виджет поддерживает светлую / тёмную / OLED темы.

**Взаимодействие со страницей:**

- Вставляет: `<div>` виджет в `document.body`
- Читает: `localStorage` (список друзей), `browser.storage.sync` (тема)
- Fetch: API LMS для получения курсов друзей

---

### `future_exams_view.js`

Показывает в аккордеоне курса предстоящие контрольные работы и экзамены.

**Как работает:**

1. Содержит захардкоженное расписание контрольных по курсам (объект `schedule`).
2. Находит текущий курс по названию в DOM.
3. Добавляет специальные `tui-accordion-item` с типом `future-exam` и датами в стиль лонгридов.
4. Поддерживает два формата отображения: `full` (название + дата) и `compact` (только дата).

**Взаимодействие со страницей:**

- Читает DOM: `cu-course-overview`, название курса
- Вставляет: `tui-accordion-item` элементы в аккордеон

### `course_exporter.js`

> ⚠️ **Отключён** (`// currently disabled`). Скрипт не выполняется при загрузке.

Планировался как инструмент для экспорта содержимого курса (материалов и задач) через API LMS. Содержит функции для запроса `/api/micro-lms/longreads/{id}/materials` и получения деталей задач.

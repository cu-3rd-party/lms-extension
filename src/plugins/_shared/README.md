# Плагин `_shared` — общие скрипты

**URL:** все страницы `https://my.centraluniversity.ru/`

Набор вспомогательных скриптов, которые загружаются на **каждой** странице LMS. Здесь живут глобальные утилиты и функции, от которых зависят остальные плагины.

---

## Файлы и их назначение

### `version_check.js`

Проверяет актуальность версии расширения.

**Как работает:**

1. При загрузке страницы делает запрос к GitHub Gist, где хранится `version.json` с актуальными версиями для каждого браузера.
2. Сравнивает версию из манифеста расширения (`browser.runtime.getManifest().version`) с удалённой.
3. Если установленная версия устарела — вставляет в шапку сайта (`ul.header__actions-list`) уведомление со ссылкой на страницу релизов GitHub.
4. Проверка повторяется каждые 30 секунд.

**Взаимодействие со страницей:**

- Читает DOM: `ul.header__actions-list`, `li[automation-id="header-action"]`
- Вставляет: клонированный `<li>` с красной ссылкой «Плагин устарел»

---

### `reset.js`

Позволяет сбросить данные расширения, сохранённые в `localStorage`.

**Как работает:**

- Подписывается на два канала сообщений:
  - `browser.runtime.onMessage` — от попапа расширения (action `RESET_LMS_LOCAL_STORAGE_FROM_POPUP`)
  - `window.addEventListener('message')` — от встроенного iframe-меню (action `RESET_LMS_LOCAL_STORAGE_IFRAME`)
- При получении команды удаляет ключи:
  - `cu.lms.actual-student-tasks-custom-filter`
  - `cu.lms.actual-student-tasks-filter`
  - `cu.lms.skipped-tasks`
  - `cu_friends_list`

**Взаимодействие со страницей:** только `localStorage`, DOM не трогает.

---

### `friends_tab.js`

Добавляет вкладку **«Друзья»** в главную навигацию.

**Как работает:**

1. Ждёт появления `tui-tabs` (навигационная панель).
2. Создаёт и вставляет ссылку `<a id="custom-friends-link">` рядом с остальными вкладками.
3. По клику открывает оверлей (`#friends-overlay`) поверх страницы с полноэкранным UI друзей.
4. UI содержит: список друзей, формы добавления/удаления, поиск по имени, отметку об общих курсах.
5. Список друзей хранится в `localStorage` (`cu_friends_list`).
6. Тема (светлая/тёмная/OLED) синхронизируется с настройками расширения через `chrome.storage.onChanged`.

**Взаимодействие со страницей:**

- Читает DOM: `tui-tabs`, `.header__tab-link`
- Вставляет: `<a>` во вкладки, `<div id="friends-overlay">` в `document.body`
- Делает fetch к API LMS для получения данных об однокурсниках

---

### `plugin_page_loader.js`

Добавляет кнопку расширения в шапку сайта и открывает по ней встроенное меню настроек.

**Как работает:**

1. Вставляет кнопку (иконку) в `ul.header__actions-list`.
2. По клику открывает полноэкранный оверлей (`#cu-plugin-overlay-container`) с двумя панелями:
   - **Левая** — `<iframe>` с попапом расширения (`popup/popup.html`), где и находятся все переключатели.
   - **Правая** — панель с контентом из GitHub Gist (новости / ченджлог).
3. При закрытии меню через `postMessage` спрашивает у iframe, были ли изменены настройки, и при необходимости перезагружает страницу.
4. Тема оверлея синхронизируется с `browser.storage.sync`.
5. По запросу из iframe может один раз перезагрузить текущую вкладку без инъекции плагинов, чтобы быстро сравнить поведение LMS с включённым и выключенным расширением.

**Взаимодействие со страницей:**

- Читает DOM: `ul.header__actions-list`
- Вставляет: кнопку в шапку, `<div>` оверлея с `<iframe>` в `document.body`
- Fetch к GitHub Gist для загрузки правой панели

### `snow.js`

Анимация падающего снега поверх страницы.

**Как работает:**

1. Читает флаг `snowEnabled` из `browser.storage.sync`.
2. Если включено — создаёт `<canvas>` с `position: fixed; z-index: 9999` и запускает анимацию 150 снежинок через `requestAnimationFrame`.
3. Подписывается на `storage.onChanged` и мгновенно включает/выключает снег при изменении настройки.

**Взаимодействие со страницей:** вставляет `<canvas>` в `document.body`, не трогает DOM самой страницы.

---

### `course_card_image_replacer.js`

Позволяет заменить стандартные иконки карточек курсов на кастомные картинки.

**Как работает:**

1. Читает из `browser.storage.local`: флаг `stickerEnabled`, URL глобального стикера (`customStickerData`) и карту «courseId → base64» (`courseIconsMap`).
2. Находит иконки курсов по селектору `img.course-icon, tui-icon[class*="course-icon"]`.
3. Заменяет `src` / `style` иконки на кастомное изображение.
4. При **режиме редактора** (включается из попапа) клик по иконке курса открывает `<input type="file">` для выбора файла — изображение сохраняется в `localStorage` для конкретного курса.
5. `MutationObserver` следит за появлением новых карточек при SPA-навигации.

**Взаимодействие со страницей:**

- Читает DOM: карточки курсов по CSS-селектору
- Заменяет: атрибуты иконок
- Хранилище: `browser.storage.local`

---

### `icons_color_fixer.js`

Исправляет цвет конкретных SVG-иконок, которые некорректно отображаются при нестандартных темах.

**Как работает:**

- Применяет CSS-фильтр `brightness(0) invert(1)` к `tui-icon`-элементам, чей атрибут `style` совпадает с именами из списка `TARGET_ICONS`.
- `MutationObserver` следит за появлением новых иконок и изменением их `style`.

**Взаимодействие со страницей:** только чтение/изменение атрибутов `<tui-icon>`, DOM не перестраивает.

---

### `debug_utils.js`

Утилита для отладки. Предоставляет глобальную функцию `cuLmsLog`, которую используют все остальные скрипты для вывода логов (можно отключить в `DEBUG_MODE`).

---

### `apricot_api.js`

- Отвечает за интерфейс в веб-странице: `window.__akhCheckApi`.
- Методы:
  - `fetchCourseDetails(courseId)`
  - `fetchTaskProgress(taskId)`
  - `fetchAllProgress()`
- Работает через `CustomEvent` и `AKH_PROXY_REQUEST`/`AKH_PROXY_RESPONSE_*`.

### `akh_bridge.js`

- Слушает `AKH_PROXY_REQUEST` и пересылает запрос в фон через `chrome.runtime.sendMessage`.
- Возвращает ответ обратно в страницу через `AKH_PROXY_RESPONSE_<id>`.
- Обрабатывает JSON из-за особенностей Firefox (строка или объект).

### `apricot_tasks_fix.js`

- Включает отображение статусов Apricot в таблице задач LMS.
- Находится в `plugins/courses/apricot_tasks_fix.js`.
- Логика:
  - Периодически (каждые 3 секунды) проходит по строкам задач
  - Нормализует названия (`normalize`): удаляет "ДЗ N", "неделя N", спецсимволы и лишние пробелы
  - Находит совпадения среди задач/групп Apricot
  - Выбирает статус с максимальным приоритетом (Rework > On review > Accepted > None)
  - Добавляет бейдж `AKH: STATUS` в столбец статуса

### `token_grabber.js`

- Берёт токены из `localStorage:
  - access: `auth-token` / `access`
  - refresh: `refresh-token` / `refresh_token` / `auth-refresh` / `refresh`
- Посылает `chrome.runtime.sendMessage({action: 'AKH_SAVE_TOKENS', access, refresh})`.

### Как ориентация

- Все эти скрипты работают вместе: `token_grabber` даёт токен, `apricot_api` - интерфейс, `akh_bridge` - прокси, `apricot_tasks_fix` - отображение.
- Цель — показать текущие статусы Apricot задач внутри LMS без переписывания интерфейса LMS.

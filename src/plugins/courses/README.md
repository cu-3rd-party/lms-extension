# Плагин `courses` — страница задач

**URL:** `/learn/tasks` (список всех задач студента)

Расширяет стандартную таблицу задач дополнительными колонками, фильтрами и возможностью помечать задания как намеренно пропущенные.

---

## Файлы и их назначение

### `tasks_fix.js`

Основной и единственный скрипт плагина. Полностью переосмысляет таблицу задач.

**Что добавляет:**

#### Колонка «Вес задания»

- Делает запрос к `/api/micro-lms/tasks/student` для получения данных о задачах.
- Добавляет в заголовок таблицы новую колонку «Вес».
- Для каждой строки задачи отображает её вес в итоговой оценке курса.

#### «Метод скипа»

- Рядом с каждым заданием появляется кнопка (иконка). По клику задание помечается как намеренно пропущенное.
- Кнопка встраивается в ячейку late days так, чтобы не выталкивать нативные LMS-действия за пределы hover-области строки.
- Статус «Метод скипа» сохраняется в `localStorage` (`cu.lms.skipped-tasks`) как Set из идентификаторов задач.
- Таким задачам ставится фиолетовый чип «Метод скипа» вместо стандартного статуса.

#### Выделение семинаров

- Семинарные задания определяются по ключевому слову в названии.
- Их строки окрашиваются в отдельный (более тёмный) цвет фона.
- Им ставится чёрный чип «Семинар».

#### Кастомные фильтры

- Добавляет строку поиска над таблицей для фильтрации задач по имени, курсу или статусу.
- Состояние фильтра сохраняется в `localStorage` (`cu.lms.actual-student-tasks-custom-filter`).

#### Сброс стандартных фильтров LMS

- При загрузке страницы автоматически сбрасывает стандартные фильтры LMS (`cu.lms.actual-student-tasks-filter`), чтобы всегда показывались все задачи без предустановленной фильтрации.

#### Встроенная замена эмодзи

- Содержит встроенную версию `emoji_swap` для корректной работы на странице задач (внешний `emoji_swap.js` не грузится на `/learn/tasks`, чтобы не конфликтовать).

#### Чип «Доработка»

- Задачи, которые были отклонены и находятся на доработке, получают красный чип «Доработка».

**Как работает технически:**

1. `MutationObserver` следит за появлением `.task-table` в DOM.
2. После появления таблицы вызывает `runLogic()`, которая: запрашивает данные API, добавляет заголовок, перебирает строки таблицы и заполняет их данными.
3. При переходе на страницу архива (`/tasks/archived-student-tasks`) все изменения откатываются через `cleanupModifications()`.

**Взаимодействие со страницей:**

- Читает DOM: `.task-table`, `tr[class*="task-table__task"]`
- Вставляет: заголовок колонки, ячейки с весом, кнопки, чипы статусов, строку фильтров
- Хранилище: `localStorage`, `browser.storage.sync`
- Fetch: `/api/micro-lms/tasks/student`
